- 14일 기준 입시 기간 상태(시작 전, 기간 중, 이후)를 자동으로 판정하고 동기화하는 로직을 구현함.
- **주요 변경 사항:**
  - **상태 판정 로직 구현:**
    - `AdmissionPeriodService`에 `getCurrentPeriodStatus` 메서드를 구현하여 현재 시점의 입시 상태(`AdmissionPhase`)와 활성 카테고리를 결정함.
    - `determinePeriodPhase` 등 6개의 헬퍼 메서드를 도입하여 복잡한 날짜 계산 로직을 분리하고 순환 복잡도(Cyclomatic Complexity)를 대폭 개선함.
  - **경계 조건 처리 강화:**
    - 입시 시작 14일 전부터 '시작 전' 상태로 활성화되도록 기준을 적용함.
    - 종료일 당일 23:59:59까지 '기간 중'으로 판정되도록 경계 조건을 수정하고, 연도 경계(12월→1월) 처리를 보완함.
  - **인터페이스 및 테스트:**
    - `AdmissionPeriodStatus` 인터페이스를 정의하여 상태 정보를 구조화함.
    - 다양한 기간 및 경계 조건 시나리오에 대한 단위 테스트 34개를 작성하여 안정성을 검증함.

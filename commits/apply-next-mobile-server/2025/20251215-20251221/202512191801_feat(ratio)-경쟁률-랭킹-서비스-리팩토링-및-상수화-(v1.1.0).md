feat(ratio): 경쟁률 랭킹 서비스 리팩토링 및 상수화

- `RatioRankingService`의 코드를 리팩토링하고 관련 상수를 모듈화하여 가독성, 유지보수성, 확장성을 향상함.
- **주요 변경 사항:**
  - **상수화:**
    - `RATIO_RANKING_CONSTANTS` 객체를 신규 생성하여 페이지네이션(`DEFAULT_LIMIT`) 및 서비스 연도(`getDefaultServiceYear`) 관련 상수를 중앙 관리함.
    - `getServiceYear` 유틸리티 함수를 `getDefaultServiceYear`로 래핑하여 사용함.
  - **Enum 도입:**
    - `RankingBasis` Enum을 도입하여 랭킹 기준('Univ', 'Major', 'Now')을 타입 안전하게 관리하고 가독성을 높임.
    - `RatioDisplayMode` Enum을 도입하여 API 응답의 `displayMode` 필드를 `RatioDisplayMode.UNIV_BASED`, `RatioDisplayMode.CLICK_BASED`와 같이 명확하게 정의함.
  - **서비스 로직 개선:**
    - `getTopUniversityRankings`, `getTopMajorRankings`, `getClickBasedRankings` 메서드에서 반복되는 카테고리 결정, 대학 ID 추출, 부가 데이터 조회 로직을 `resolveCategory`, `extractUnivIds`, `fetchUnivSupplementaryData`와 같은 private 헬퍼 메서드로 분리하여 중복을 제거하고 비즈니스 로직에 집중함.
    - 메서드 시그니처에 기본값을 상수로 대체하고, 타입 힌트를 `AdmissionCategory`로 구체화함.
  - **DTO 개선:**
    - `RatioRankingListResponseDto`의 `displayMode` 필드를 `RatioDisplayMode` Enum 타입으로 변경하여 타입 안전성을 확보하고, Swagger 문서에 Enum 값을 표시함.
- **파일 구조 개선:**
  - `enums` 및 `constants` 디렉토리에 새로운 Enum과 상수 파일을 추가하고, `index.ts`를 통해 모듈의 가시성을 높임.

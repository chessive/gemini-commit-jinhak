refactor(statistics): 일자별 통계 UI 개선 및 안정성 강화

일자별 통계 위젯(`DailyStatisticWidget`)의 차트 가독성을 높이고 코드 품질을 개선하기 위해 UI 레이아웃과 내부 로직을 전면 리팩토링함.

### 주요 변경 사항

- **1. 차트 중심의 레이아웃 재구성:**
  - **차트 영역 대폭 확대 (+71%):** KPI 카드와 날짜 필터 영역의 패딩, 아이콘, 폰트 크기 등을 최적화하여 차트가 차지하는 수직 공간을 기존 `~308px`에서 `~527px`로 약 71% 확대함.
  - **전체 위젯 높이 증가:** 위젯의 전체 높이를 화면의 70%에서 82%로 늘려, 차트가 더 많은 공간을 확보하도록 함.
  - **필터 인라인 배치:** 날짜 선택 필드와 빠른 필터 버튼을 `Row`로 재배치하여 수직 공간을 추가로 확보하고 UI를 컴팩트하게 개선함.

- **2. 코드 품질 및 유지보수성 향상:**
  - **상수화:** `0.82`, `12.0` 등 하드코딩된 수치들을 `WIDGET_HEIGHT_RATIO`, `COMPONENT_SPACING` 등의 명시적인 상수로 대체하여 가독성과 유지보수성을 높임.
  - **네이밍 개선:** `WHEN_DAY_START_END`와 같이 의미가 모호했던 상수명을 `DEFAULT_DATE_RANGE_DAYS`로 변경하여 코드의 의도를 명확히 함.
  - **미사용 파라미터 제거:** `StatisticsKpiCard`에서 사용되지 않는 `minConsults` 파라미터를 제거하여 코드를 간결화함.

- **3. 안정성 및 성능 강화:**
  - **날짜 범위 검증:** 날짜 조회 기간을 최대 365일로 제한하는 검증 로직을 추가하여, 과도한 데이터 조회로 인한 성능 저하를 방지하고 사용자에게 안내 메시지를 표시하도록 함.
  - **애니메이션 중복 방지:** 날짜 필터 변경 시, 애니메이션이 실행 중일 경우 중복 호출을 방지하는 `_isAnimating` 플래그를 추가하여 UI 안정성을 높임.

- **4. 접근성 개선:**
  - `Semantics` 위젯을 KPI 카드, 날짜 필터, 필터 버튼 등에 적용하여 스크린 리더 사용자를 위한 컨텍스트(예: "총 상담수 42명", "최근 7일 선택됨")를 명확하게 제공하도록 접근성을 강화함.

### 기대 효과

- **가독성 향상:** 차트 영역이 대폭 확대되어 사용자가 통계 데이터를 더 명확하고 쉽게 인지할 수 있음.
- **사용자 경험 개선:** 컴팩트하고 정돈된 UI와 안정적인 애니메이션, 유효성 검증을 통해 전반적인 사용자 경험이 향상됨.
- **코드 품질 향상:** 상수화, 네이밍 개선, 접근성 강화를 통해 코드의 유지보수성과 품질이 높아짐.

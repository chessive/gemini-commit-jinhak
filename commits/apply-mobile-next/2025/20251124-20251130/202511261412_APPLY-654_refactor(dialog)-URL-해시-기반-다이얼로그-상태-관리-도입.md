[APPLY-654] refactor(dialog): URL 해시(#) 기반 다이얼로그 상태 관리 도입

- Clarity 히트맵에서 메인 화면과 다이얼로그를 별도 페이지로 분리하여 분석할 수 있도록, 다이얼로그 상태를 URL 해시와 연동함.
- **주요 변경 사항**:
  - `dialog.store.ts`: `openDialog` 시 URL에 `#` 해시를 추가하고, `closeDialog` 시 해당 해시만 선택적으로 제거하는 로직을 구현함.
  - `dialog-provider.tsx`: `popstate` 이벤트 리스너를 추가하여 브라우저의 뒤로가기/앞으로가기 버튼으로 다이얼로그를 제어할 수 있도록 개선함.
- **기대 효과**:
  - Clarity에서 URL 기반으로 히트맵 스크린샷이 분리되어 분석 정확도 향상.
  - 브라우저 뒤로가기 버튼으로 다이얼로그를 닫는 직관적인 UX 제공.

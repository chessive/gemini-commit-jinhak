refactor(student-info, ui): 학생 정보 화면 UI, 성적 입력 화면과 완벽히 통일

사용자 경험의 일관성을 확보하기 위해, `StudentInfoScreen`의 레이아웃과 스타일을 `CalculateScreen`의 디자인에 맞춰 전면 리팩토링함.

### 주요 변경 사항

#### 1. 레이아웃 구조 통일
- **위젯 구조 단순화**: `StudentInfoForm`의 불필요한 7단계 중첩 구조를 4단계로 단순화하고, 장식용 `Border`와 `ClipRect` 등을 제거하여 코드 가독성 및 렌더링 효율을 개선함.
- **정렬 및 간격 조정**: 기존의 중앙 정렬, 가변 너비 구조를 `CalculateScreen`과 동일한 왼쪽 정렬, 고정 너비(`whiteBoxWidth * 0.9`) 방식으로 변경하여 두 화면 간의 시각적 흐름을 일치시킴.
- **공간 활용 최적화**: `StudentInfoScreen`의 불필요한 상/하단 간격(`SizedBox`)을 제거하여 폼 영역을 최대로 활용하도록 구조를 변경함.

#### 2. 스타일 완전 동기화 (`DynamicFieldStyles`)
- **스타일 값 통일**: `CalculateScreen`의 스타일에 맞춰 `DynamicFieldStyles`의 모든 값을 수정함. (e.g., `borderRadius`: 12.0→4.0, `focusColor`: blue→black, `fontWeight`: 500→700)
- **반응형 폰트 지원**: `labelStyle`과 `inputStyle`을 `BuildContext`를 받는 정적 메서드로 변경하여, 화면 높이에 비례하는 동적 폰트 크기를 적용함.

#### 3. 로딩 및 에러 UI 개선
- **디자인 통일**: 로딩 인디케이터에 브랜드 색상(`0xff2b87ec`)을 적용하고, 에러 화면의 아이콘, 텍스트, 버튼 스타일을 개선하여 앱 전반의 피드백 UI와 일관성을 맞춤.

### 기대 효과
- **일관된 사용자 경험**: 두 핵심 화면 간의 레이아웃 및 스타일이 동일해져, 사용자가 화면을 전환할 때 느끼는 시각적 이질감을 완전히 해소함.
- **코드 품질 향상**: 불필요한 위젯 중첩을 제거하고 레이아웃 로직을 단순화하여 코드의 가독성과 유지보수성을 높임.
- **디자인 완성도**: 앱의 핵심 기능인 두 화면의 디자인 통일성을 확보하여 전체적인 디자인 완성도를 개선함.

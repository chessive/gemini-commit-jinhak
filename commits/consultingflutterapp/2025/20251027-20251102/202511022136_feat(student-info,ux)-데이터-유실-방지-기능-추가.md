feat(student-info, ux): 학생 정보 입력 취소 시, 데이터 유실 방지 기능 추가

학생 정보 입력 화면에서 사용자가 데이터를 입력한 후 '이전' 버튼을 클릭할 경우, 데이터가 유실되는 것을 방지하기 위한 확인 절차를 추가함.

### 주요 변경 사항

#### 1. '이전' 버튼 로직 수정 (`StudentInfoScreen`)
- `onPrevious` 콜백을 `async` 함수로 변경하여 다이얼로그의 사용자 응답을 기다릴 수 있도록 함.

#### 2. 데이터 입력 여부 확인 로직 추가
- 폼에 입력된 데이터가 있는지 (`formData.data.isNotEmpty`) 먼저 확인.
- 입력된 내용이 없으면 사용자에게 불필요한 질문 없이 즉시 이전 화면으로 이동하여 사용자 경험(UX)을 개선함.

#### 3. 확인 다이얼로그 표시
- 입력된 데이터가 존재할 경우, 기존의 `showConfirmDialog` 유틸리티를 호출하여 사용자에게 데이터가 삭제됨을 알리고 취소 의사를 재확인함.

#### 4. 데이터 초기화 로직 연동
- 사용자가 다이얼로그에서 '예'를 선택하면, `consultingStore.resetFormData()`와 `foundationStore.clearConsultHistory()`를 순차적으로 호출하여 관련 상태를 모두 초기화함. 이를 통해 다음 세션에서 데이터가 남는 문제를 방지하고 데이터 무결성을 확보함.

### 기대 효과
- **데이터 보호**: 사용자의 실수로 인한 데이터 손실 위험을 방지함.
- **사용자 경험 개선**: 명시적인 확인 절차를 통해 사용자에게 신뢰를 주고, 불필요한 확인 단계를 제거하여 편의성을 높임.
- **데이터 무결성 확보**: 다음 상담 시작 시 항상 깨끗한 상태를 보장함.

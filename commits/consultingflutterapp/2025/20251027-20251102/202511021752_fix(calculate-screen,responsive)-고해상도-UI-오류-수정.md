fix(calculate-screen, responsive): 성적 입력 화면, 고해상도 반응형 UI 오류 수정

고해상도(QHD, 4K) 디스플레이에서 발생하던 성적 입력 화면의 레이아웃 오버플로우 및 UI 깨짐 문제를 해결함.

### 문제 원인
- 화면 너비에 비례하여 UI 요소 크기를 계산하는 기존 방식이 상위 컨테이너(`whiteBoxWidth`)의 최대 너비 제약을 초과하여 오버플로우를 발생시킴.
- 고해상도에서 키보드, 레이블 등의 UI 요소가 기능적 필요 이상으로 과도하게 커지는 사용성 문제가 있었음.

### 해결 방안

#### 1. 브레이크포인트 및 최소/최대 제약 도입
- 화면 너비를 여러 구간(1280, 1920, 2560px)으로 나누는 브레이크포인트를 도입하여, 해상도별로 차등적인 크기 비율을 적용함.
- 모든 핵심 UI 요소(키보드, 점수 레이블, 토글 버튼 등)에 `.clamp()`를 사용하여 최소/최대 너비 및 높이 제약을 추가함. 이를 통해 UI 요소가 비정상적으로 커지거나 작아지는 것을 방지함.

#### 2. 오버플로우 원천 차단
- `calculate_screen.dart`의 폼 너비 계산 방식을 변경함. 화면 전체 너비(`screenWidth`)가 아닌, 실제 사용 가능한 부모 컨테이너 너비(`whiteBoxWidth`)에서 키보드 너비를 빼는 방식으로 수정하여 오버플로우를 원천적으로 차단함.

#### 3. 매직 넘버 상수화 (`CalculateScreenConstants`)
- 반응형 레이아웃 계산에 사용되는 모든 수치(비율, 제약 값, 브레이크포인트)를 `CalculateScreenConstants.dart` 파일로 분리하여 중앙에서 관리함.
- 이를 통해 코드의 가독성과 유지보수성을 크게 향상시킴.

### 기대 효과
- **레이아웃 안정성**: 4K를 포함한 모든 해상도에서 오버플로우 없이 안정적인 레이아웃을 보장함.
- **사용성 개선**: 화면 크기와 관계없이 모든 UI 요소가 일관되고 최적화된 크기로 표시됨.
- **코드 품질 향상**: 하드코딩된 매직 넘버를 제거하고 중앙화된 상수를 사용하여 코드의 명확성과 유지보수성을 개선함.

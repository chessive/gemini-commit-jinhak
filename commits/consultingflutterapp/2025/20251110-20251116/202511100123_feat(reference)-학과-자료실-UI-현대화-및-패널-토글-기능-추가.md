feat(reference): 학과 자료실 UI 현대화 및 패널 토글 기능 추가

학과 자료실(`SubjectReferenceScreen`)의 UI를 최근 통계 화면에 적용된 중앙 디자인 시스템에 맞춰 현대화하고, PDF 뷰어 공간을 극대화할 수 있는 패널 토글 기능을 추가하여 사용자 경험을 개선함.

### 주요 변경 사항

- **1. 디자인 시스템 통합 및 UI 현대화:**
  - **중앙 색상 시스템 적용:** 기존에 하드코딩되어 있던 10개의 색상 값을 모두 `ChartColorPalette`를 사용하도록 변경하여, 통계/결과 화면과 완벽한 시각적 일관성을 확보함.
  - **일관된 여백 시스템 적용:** 하드코딩된 패딩, 높이 등 5개의 레이아웃 값을 `Spacing` 상수로 대체하고, `searchContainerHeight` 등 신규 상수를 추가하여 8px 그리드 시스템을 준수하도록 함.
  - **컴포넌트 스타일 통일:** 검색 아이콘(18px), 폰트 크기(15px) 등 UI 요소들을 앱의 다른 부분과 동일한 스타일로 표준화함.

- **2. 패널 토글 기능 추가 (UX 개선):**
  - **슬라이드 패널 구현:** `AnimationController`를 사용하여 우측 학과 목록 패널을 부드럽게 열고 닫을 수 있는 기능을 추가함. 이를 통해 사용자가 필요에 따라 PDF 뷰어 공간을 25% 더 넓게 활용할 수 있게 됨.
  - **지능형 토글 버튼:**
    - 패널이 열려 있을 때는 검색창 우측에 '닫기'(`Icons.menu_open`) 아이콘이 표시됨.
    - 패널이 닫혔을 때는 화면 우측 하단에 FAB(플로팅 액션 버튼) 스타일의 '열기'(`Icons.menu`) 아이콘이 나타나도록 하여, PDF 뷰어 컨트롤과 겹치지 않으면서도 직관적인 UX를 제공함.

- **3. 레이아웃 및 코드 품질 개선:**
  - **`TextField` Overflow 해결:** `label` 속성 대신 `hintText`와 `prefixIcon`을 사용하는 방식으로 `TextField`를 리팩토링하여, 특정 조건에서 발생하던 렌더링 오버플로우 문제를 해결함.
  - **`Stack` 레이아웃 적용:** 플로팅 버튼의 위치를 정확하게 제어하기 위해 기존 `Row` 구조를 `Stack` 기반으로 재구성함.
  - **디자인 시스템 확장:** `theme.dart`에 `selectedTileColor`를 추가하는 등, 이번 리팩토링을 통해 중앙 디자인 시스템을 더욱 확장하고 견고하게 만듦.

### 기대 효과

- **향상된 사용자 경험:** 사용자가 PDF 문서를 더 넓은 화면에서 집중해서 볼 수 있게 됨.
- **디자인 일관성:** 앱 전반에 걸쳐 통일된 디자인 언어를 사용하여 전문적이고 안정적인 인상을 줌.
- **유지보수성 증대:** 중앙화된 디자인 시스템 덕분에 향후 UI 변경 및 테마 적용이 매우 용이해짐.
